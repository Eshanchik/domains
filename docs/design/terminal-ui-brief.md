# Бриф для Claude Design — визуальный стиль «Terminal UI / CLI Aesthetics / Matrix»

> Это самодостаточный промт-задание для Claude Design. Его можно отдать целиком.
> Цель — **перекрасить** существующий интерфейс DomainGuard в эстетику
> терминала/командной строки/Matrix, **не меняя** информационную архитектуру,
> семантику разметки, тексты, роли и поведение. Это **скин поверх** имеющихся
> шаблонов и слоя компонентов, а не переписывание приложения.

---

## 1. Контекст продукта

**DomainGuard** — внутренняя система мониторинга доменов: реестр доменов
нескольких компаний и их проектов, контроль сроков истечения (RDAP/WHOIS),
SSL-сертификатов, репутации (VirusTotal), health-check URL, учёт стоимости
продлений и адресные алерты (Telegram/Slack/Discord). Аудитория — технические
администраторы. «Терминальная» эстетика органична: пользователь и так живёт в
консоли.

**Стек (не менять):** серверный рендеринг Jinja2 + HTMX + Tailwind (Play CDN,
без шага сборки). Никаких SPA/React. Тёмная тема — основная; сейчас есть
переключатель темы. Тексты интерфейса — **русские**, технические токены (статусы,
типы проверок) — латиницей.

**Что уже есть (на это и накладываем скин):** дизайн-система вынесена в
Tailwind-слой компонентов в `templates/base.html`, макросы — в
`templates/_components.html`. Перекраска должна происходить **на уровне этих
классов**, чтобы страницы не переписывались:

| Класс/макрос | Роль |
|---|---|
| `.card`, `.card-p` | панель/поверхность |
| `.page-title`, `.section-title` | заголовки |
| `.btn` + `.btn-primary/secondary/ghost/danger`, `.btn-sm` | кнопки |
| `.input`, `.select`, `.textarea`, `.label`, `.checkbox` | формы |
| `.badge` + `.badge-gray/green/red/amber/blue/indigo` | пилюли/статусы |
| `.flash` + `.flash-error/success/info` | баннеры |
| `.dg-table` (+ `thead th`, `tbody td`) | таблицы |
| `.stat`, `.stat-value`, `.stat-label` | плитки дашборда |
| `.nav-link`, `.nav-link.active`, `.nav-section` | боковое меню |
| `ui.icon`, `ui.badge`, `ui.status_badge`, `ui.page_header`, `ui.empty_row` | макросы |

**Задача-максимум:** переопределить эти классы (цвета, шрифт, рамки, эффекты) так,
чтобы всё приложение выглядело как единый терминал — без правки бизнес-разметки.

## 2. Философия стиля

- **Консоль, а не «приложение».** Экран — это большое окно терминала. Контент —
  вывод команд: колоночные таблицы, лог-строки, ASCII-панели.
- **Моноширинность во всём.** Один моноширинный шрифт формирует ритм и «сетку
  символов».
- **Фосфор на near-black.** Зелёный люминофор (или янтарь) на почти чёрном фоне;
  лёгкое свечение текста (`text-shadow`), как у CRT.
- **Клавиатура-first.** Явные фокус-рамки, `>`-курсор, подсказки-хоткеи.
- **Скупость и плотность.** Мало «воздуха», прямые углы или ASCII-рамки вместо
  скруглений и теней, никаких градиентов кроме matrix-фона.
- **Живость дозирована.** Мигающая каретка, matrix-дождь и scanline — тонко и
  **только** при `prefers-reduced-motion: no-preference`.

Настроение-референсы: cool-retro-term, htop/btop, tmux, dashboards в стиле
«Fallout Pip-Boy / Matrix / hacker HUD».

## 3. Палитра (дизайн-токены)

Основная тема — **green phosphor on black**. Все цвета — CSS-переменные, чтобы
можно было держать вариант «amber».

```
/* Фон/поверхности */
--term-bg:        #05080a;  /* фон экрана (near-black, чуть зелёный) */
--term-panel:     #0a0f0c;  /* панели/карточки */
--term-panel-2:   #0e150f;  /* полосатость строк, hover */
--term-border:    #1c2a20;  /* рамки/линии (тусклый зелёный) */
--term-grid:      #12331f;  /* сетка/скан-линии */

/* Текст (фосфор) */
--term-fg:        #b7ffcf;  /* основной текст */
--term-fg-dim:    #5f8f6f;  /* вторичный/muted */
--term-fg-faint:  #35513f;  /* подписи, плейсхолдеры */
--term-bright:    #33ff66;  /* акцент/интерактив (яркий фосфор) */
--term-bright-2:  #00ff41;  /* «matrix» hover/фокус */

/* Семантика статусов */
--term-ok:        #33ff66;  /* [OK]  успех/up/valid */
--term-warn:      #ffb000;  /* [WARN] amber, скоро/stale/medium */
--term-fail:      #ff3b3b;  /* [FAIL] ошибка/down/high */
--term-info:      #35c3ff;  /* [INFO] нейтральная подсветка */

/* Свечение */
--term-glow:      0 0 6px rgba(51,255,102,.35);
```

**Вариант «amber»** (переключатель темы вместо light/dark): `--term-fg:#ffd08a`,
`--term-bright:#ffb000`, свечение янтарное; фон тот же near-black. Семантику
(ok/warn/fail) сохраняем цветной, чтобы статусы читались.

**Контраст:** основной и яркий текст на фоне дают высокий контраст (AA/AAA).
`--term-fg-faint` использовать только для декоративных подписей, не для смысловой
информации, иначе не пройдёт AA.

## 4. Типографика

- **Шрифт:** моноширинный. Стек: `"JetBrains Mono", "IBM Plex Mono", "Fira Code",
  ui-monospace, "SFMono-Regular", Menlo, Consolas, monospace`. Если разрешён
  внешний шрифт — self-host JetBrains Mono (с лигатурами `calt`, `ss01`); если
  нужен офлайн/CSP — только системный `ui-monospace`. **Один** шрифт на весь UI.
- **Размеры:** база 13–14px, компактный line-height ~1.45. Заголовки не крупнее
  16–18px — «крупность» задаётся не кеглем, а UPPERCASE + `letter-spacing`.
- **Регистр:** метки, заголовки таблиц, названия секций — `UPPERCASE` с
  `letter-spacing: .08em`. Данные — как есть (домены — обязательно моно).
- **Лигатуры:** включить в декоративных местах (промпт, код), можно отключать в
  табличных данных ради выравнивания.

## 5. Раскладка и «оконность»

- Весь `<body>` — «терминал»: сплошной near-black фон, опционально тонкая
  внешняя рамка экрана (1px `--term-border`) и «титлбар» сверху со статус-строкой.
- **Боковое меню** (`.nav-link` / `.nav-section`) — список команд: каждый пункт
  с префиксом `> `; активный пункт — инверсия (фон `--term-bright`, текст фона)
  или яркий `>`-маркер и подсветка. Секции (`КАТАЛОГ`, `АДМИНИСТРИРОВАНИЕ`,
  `ЛИЧНОЕ`) — как комментарии: `# КАТАЛОГ` тусклым.
- **Топбар** — статус-строка: слева путь как «prompt» (`dg@domainguard:~/domains$`),
  справа — пользователь/роль (`user=admin role=admin`), переключатель темы
  (`[green]`/`[amber]`), выход (`[exit]`).
- **Панели/карточки** (`.card`) — ASCII-рамки box-drawing: `┌─ TITLE ───────┐`
  сверху, `│ … │` по бокам, `└───────────────┘` снизу; либо простая рамка 1px
  `--term-border` с титлом в стиле `┤ TITLE ├`. Без теней и скруглений.
- **Плотность:** компактные паддинги; разделители строк — 1px `--term-border`
  или чередование `--term-panel`/`--term-panel-2` («zebra» как в `ls -l`).

## 6. Компоненты (переопределение классов)

- **Кнопки** (`.btn`): вид `[ LABEL ]` — квадратные, рамка 1px, моно, UPPERCASE.
  - `.btn-primary` — заливка `--term-bright`, текст `--term-bg` (инверсия),
    свечение при hover.
  - `.btn-secondary` — прозрачная, рамка `--term-border`, текст `--term-fg`;
    hover — инверсия.
  - `.btn-ghost` — без рамки, только текст; hover — подсветка.
  - `.btn-danger` — рамка/текст `--term-fail`, hover-инверсия красного.
  - Фокус — двойная яркая рамка (`outline: 1px solid var(--term-bright)`).
- **Поля ввода** (`.input/.select/.textarea`): фон `--term-panel`, рамка
  `--term-border`, текст `--term-fg`, **мигающая каретка** (`caret-color:
  var(--term-bright)`), плейсхолдер `--term-fg-faint`. Префикс `>` перед
  активным текстовым полем (декоративный). Фокус — рамка `--term-bright` + glow.
  Чекбокс (`.checkbox`) — как `[ ]`/`[x]` (можно кастом-рендер через `appearance:
  none` + `::after`).
- **Бейджи/статусы** (`.badge`, `ui.status_badge`): в скобках, моно, UPPERCASE:
  `[OK]` (green), `[WARN]` (amber), `[FAIL]` (red), `[STALE]` (amber),
  `[HIGH]/[MED]/[LOW]` для severity, `[INFO]` (info). Без «пилюльной» скруглённой
  формы — прямоугольные или вовсе только цветной текст в скобках. Маппинг
  `ui.status_badge`: `ok/up→[OK]`, `fail/down/high→[FAIL]`, `stale/medium→[WARN]`,
  `low→[LOW]`.
- **Таблицы** (`.dg-table`): вывод CLI. Заголовок — UPPERCASE тусклым
  (`--term-fg-dim`) с нижней линией `--term-border`. Строки — моно, zebra,
  hover = инверсия строки (фон `--term-bright`/тусклый, текст контрастный).
  Сортируемые заголовки — со стрелкой `▲/▼`. Ячейка домена — яркий фосфор-линк.
- **Плитки дашборда** (`.stat`): мини-панели с рамкой; крупное моно-число
  (`--term-bright`, glow), подпись UPPERCASE тусклым; «опасные» значения —
  `--term-fail`. Можно добавить ASCII-бар/индикатор (`▓▓▓░░ 62%`).
- **Меню «⋮»** (dropdown в строках доменов): триггер `[…]` или `⋮`; панель —
  ASCII-рамка, пункты с `>`-префиксом, hover-инверсия. Сохранить как нативный
  `<details>` (без внешнего JS).
- **Флеш-баннеры** (`.flash`): строки лога — `!! ` для error (red), `>> ` для
  success (green), `-- ` для info; рамка слева цветная.
- **Пустые состояния** (`ui.empty_row` и пр.): `-- нет данных --` / `∅ пусто`
  тусклым по центру; **тексты не менять** (напр. «Домены не найдены.»,
  «Активных алертов нет.»).
- **Пагинация:** `[ ‹ ]` `[ 1 ]` `[ 2 ]` `[ › ]`, активная страница — инверсия.
- **Иконки** (`ui.icon`): заменить на моно-глифы/скобочные маркеры или оставить
  тонкие линейные иконки в цвете `currentColor` (фосфор). В меню — префикс `>`.

## 7. Мотивы и эффекты (все — с уважением к `prefers-reduced-motion`)

- **Мигающая каретка** `█` в промптах/заголовках и в фокусе полей (CSS
  `@keyframes blink`).
- **Matrix rain** — фон для страницы входа и, опционально, для пустых состояний:
  падающие катакана/латиница-глифы, очень тускло (позади контента, `opacity ~.08`,
  `pointer-events:none`, `z-index:-1`), с виньеткой. Реализация — маленький
  инлайновый `<canvas>`/CSS; **отключать** при reduced-motion и на мобильных
  (перф). Не мешать читаемости.
- **Scanline / CRT** — очень тонкая горизонтальная линовка (`repeating-linear-
  gradient`, `opacity ~.04`) и лёгкая виньетка по краям экрана. Опционально,
  выключаемо, off при reduced-motion.
- **Свечение текста** — `text-shadow: var(--term-glow)` на ярких акцентах
  (числа, заголовки, активные пункты). Умеренно, чтобы не «плыло».
- **Typewriter reveal** — опционально для заголовка страницы/логина (печать по
  символу); off при reduced-motion; не задерживать доступ к контенту.

---

## 8. Состояния и доступность

- **Фокус:** всегда видимая рамка `--term-bright` (не убирать `outline`).
- **Hover:** инверсия для строк/кнопок/пунктов меню.
- **Disabled:** тусклый `--term-fg-faint`, без свечения.
- **Selection:** `::selection` — фон `--term-bright`, текст `--term-bg`.
- **Контраст:** проверить AA для всех текст/фон пар; не выносить смысловую
  информацию в `--term-fg-faint`.
- **Движение:** весь декоративный motion — под `@media (prefers-reduced-motion)`.
- **Клавиатура:** порядок табуляции, доступность `<details>`-меню, видимый фокус.
- **Семантика:** сохранить теги/структуру, `aria-*`, роли; скин — только стили.

---

## 9. Жёсткие ограничения (не ломать)

- **Не менять** маршруты, `action`/`method` форм, `name`/`value` полей, HTMX-
  атрибуты (`hx-*`) и поведение (авто-сабмит фильтров, kebab-действия, резолв
  алерта, «Проверить сейчас»).
- **Не менять русские тексты** и латинские токены статусов/типов.
- **Не трогать** маскирование секретов (`***`, `type="password"`) — показывать
  как `••••••` максимум декоративно, но значения не раскрывать.
- **Сохранить** роли/скоуп-гейтинг разметки, пагинацию, экспорт CSV.
- **Стек:** остаёмся на Tailwind-слое компонентов + `<style
  type="text/tailwindcss">`; без сборки, без React. Переключатель темы
  переиспользовать (green ↔ amber вместо light ↔ dark).

## 10. Экраны (терминальный маппинг)

- **Вход (`/login`)** — «загрузка терминала»: matrix-фон, псевдо-boot-строки
  (декоративно), форма как промпт: `login:` / `password:` с мигающей кареткой,
  кнопка `[ ВОЙТИ ]`. При включённой 2FA — поле `2fa code:`.
- **Обзор (`/`)** — HUD: ряд `.stat`-панелей как «датчики» (крупные моно-числа со
  свечением, опц. ASCII-бары), ниже панель «ПО КОМПАНИЯМ» как CLI-таблица.
  «Опасные» метрики — красным.
- **Домены (`/domains`)** — центральный экран: строка фильтров как набор `--flag`
  (селекты авто-применяются), таблица-вывод `ls`-стиля с колонками
  DOMAIN / PROJECT / EXPIRES / SSL / AUTO-RENEW / ACTIVE, SSL как `[OK]/[WARN]/
  [FAIL]`, действие в строке — `[…]`-меню (Открыть/Изменить/Проверить сейчас/
  Архив). Bulk — как выбор строк + командная панель снизу.
- **Карточка домена (`/domains/{id}`)** — «dossier»: блок полей как `key: value`
  моно-список; секции Health-checks / Алерты / Платежи / Проверки / История — как
  отдельные ASCII-панели с таблицами; формы добавления — раскрывающиеся
  (`<details>`) промпты.
- **Алерты (`/alerts`)** — лог событий: строки как записи журнала
  `[HIGH] ssl domain.com  2026-07-21 12:00 ›`, вся строка кликабельна.
- **Детали алерта (`/alerts/{id}`)** — «event dump»: заголовок с типом, панель
  `key: value` (домен-ссылка, severity `[HIGH]`, состояние `[ACTIVE]/[RESOLVED]`,
  времена), разбор `payload_json` как список, таблица последних проверок; кнопка
  `[ РЕЗОЛВ ]` (Manager+).
- **Расходы (`/costs`)** — сводки и прогноз как отчётные таблицы/бар-чарты из
  ASCII-символов.
- **Импорт (`/import`)** — «paste & run»: textarea как ввод, предпросмотр —
  таблица-diff (`created/updated/error` → `[NEW]/[UPD]/[ERR]`).
- **Каталог (Компании/Проекты/Теги)** — простые CLI-таблицы с `[ + ]`-действием.
- **Пользователи/Каналы/Регистраторы/Настройки** — таблицы + формы; секреты
  как `••••` / `[ SET ]`; статус теста канала — `[OK]/[FAIL]`.
- **2FA / API-токены / Вебхуки** — промпт-формы; одноразовый токен/секрет — в
  ASCII-рамке «скопируйте один раз», моно, с кнопкой копирования.

## 11. Формат результата от Claude Design

Ожидаемая поставка — **тема-скин**, которую можно вложить в существующий
`base.html`:

1. Обновлённый `tailwind.config` (палитра `term-*`, mono-шрифт) и блок
   `<style type="text/tailwindcss">` с переопределением слоя компонентов
   (`.card`, `.btn-*`, `.badge-*`, `.input`, `.dg-table`, `.stat`, `.nav-link`,
   `.flash`, `.page-title`, `.section-title`).
2. Небольшой инлайновый CSS/JS для эффектов (каретка, matrix-canvas, scanline) —
   всё под `prefers-reduced-motion`, без внешних зависимостей и сетевых запросов
   (или self-host шрифта).
3. Правки макросов `ui.badge`/`ui.status_badge` (обёртка `[…]`) и `ui.icon`
   (глифы), если нужно — но **без** изменения сигнатур и вызовов на страницах.
4. Оба варианта темы (green/amber) через переключатель, значение в `localStorage`.

**Критерий приёмки:** ни один существующий шаблон страницы (кроме `base.html` и
`_components.html`) не требует правок; все 20 страниц выглядят как единый
терминал; тексты, маршруты, роли и поведение не изменились; движение уважает
`prefers-reduced-motion`; контраст проходит AA.

## Приложение A. ASCII-скетчи (ориентир, не пиксель-перфект)

Вход:

```
┌────────────────────────────────────────────┐
│  DOMAINGUARD // secure domain monitor       │
│                                             │
│  booting terminal.......... [OK]            │
│                                             │
│  login:    admin▮                           │
│  password: ••••••••                         │
│                                             │
│           [ ВОЙТИ ]                          │
└────────────────────────────────────────────┘
        · matrix rain на фоне, тускло ·
```

Список доменов:

```
dg@domainguard:~/domains$ ls --filter                       user=admin role=admin  [green] [exit]
# КАТАЛОГ
> ОБЗОР   > ДОМЕНЫ   > АЛЕРТЫ   > РАСХОДЫ

┌─ ДОМЕНЫ (2) ───────────────────────────────────────────────────────  [ + ДОБАВИТЬ ] ┐
│ [q______]  company▾  project▾  tag▾  expires▾  [x] archived                          │
│                                                                                      │
│  DOMAIN               PROJECT     EXPIRES      SSL      AUTO-RENEW  ACTIVE            │
│  everness.online      ACME Web    2026-11-17   [WARN]   да          [OK]    [ … ]     │
│  forgeofreason.com    ACME Web    2026-08-09   [FAIL]   неизвестно  [OK]    [ … ]     │
│                                                                                      │
└──────────────────────────────────────────────────────────────────────────────────┘
```

Меню строки `[ … ]`:

```
┌───────────────────┐
│ > Открыть         │
│ > Изменить        │
│ > Проверить сейчас│
│ > В архив         │
└───────────────────┘
```
